# FAS (Fully Automation System) 교차 검증 프롬프트

## 프로젝트 요약

FAS는 Mac Studio에서 24시간 무중단 가동되는 AI 워커 시스템입니다. 캡틴(Captain)이 메인 워커로서 n8n 오케스트레이션을 통해 코딩, 문서화, 분석, 자동화 업무를 수행하고, 헌터(Hunter)는 격리된 외부 머신에서 브라우저 기반 작업만 담당합니다. Express 기반 Gateway가 캡틴-헌터 간 태스크 통신을 중개하며, Telegram/Slack/Notion을 통해 사용자에게 보고합니다.

## 검증 항목

아래 각 항목에 대해 **[안전/주의/위험]** 등급과 **구체적 근거**를 제시해주세요.

### 1. 코드 품질
- 타입 안전성 (TypeScript strict mode, any 사용 여부)
- 에러 핸들링 패턴의 일관성
- 함수 단일 책임 원칙 준수
- 코드 중복 여부
- 테스트 커버리지 및 테스트 품질

### 2. 아키텍처
- 캡틴-헌터 간 통신 구조의 적절성
- 모듈 간 의존성 방향 (순환 의존 여부)
- Gateway의 단일 장애점(SPOF) 가능성
- 확장성 고려 여부

### 3. 보안
- 헌터 격리 프로토콜의 실효성
- API 인증/인가 메커니즘
- Rate limiting 구현의 적절성
- 입력 검증 (sanitizer)의 완전성
- 시크릿 관리 방식

### 4. 문서-코드 일치
- CLAUDE.md, devspec.md, PLAN.md 등 문서가 실제 코드와 일치하는지
- 문서에는 있으나 미구현된 기능
- 구현되어 있으나 문서화되지 않은 기능

### 5. 운영 안정성
- 24시간 무중단 가동을 위한 에러 복구 메커니즘
- 로깅 및 모니터링 구현
- Docker/인프라 설정의 적절성

### 6. 알림 시스템
- Telegram/Slack/Notion 알림 라우팅 로직
- 알림 실패 시 fallback 처리
- 알림 패턴 ([APPROVAL_NEEDED], [BLOCKED] 등) 감시 로직

## 종합 평가 요청

위 6개 항목을 종합하여:
1. 전체 시스템의 **성숙도 등급** (Prototype / Alpha / Beta / Production-ready)
2. **즉시 개선이 필요한 상위 3개 이슈**
3. **잘 설계된 상위 3개 요소**
4. **다음 단계로 권장하는 작업** (우선순위순)

을 정리해주세요.
